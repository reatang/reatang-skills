# Shannon Thinking

Claude Shannon's six creative thinking techniques from his 1952 Bell Labs talk. Apply these systematically when stuck on complex problems.

## The Three Foundations

Before applying techniques, verify these prerequisites exist:

1. **Training & Experience** — Domain knowledge is non-negotiable
2. **Intelligence** — Sufficient cognitive capacity for the problem domain
3. **Motivation** — Curiosity + constructive dissatisfaction + pleasure in results

If motivation is lacking, no technique will help. The drive to find answers must come first.

## Six Creative Thinking Techniques

Apply these iteratively. Not all will apply to every problem.

### 1. Simplification

Strip the problem to its essentials.

- Remove all extraneous data and constraints
- Solve the simplified version first
- Add refinements back incrementally

**Question:** What is the core of this problem if I remove everything non-essential?

### 2. Similar Known Problems

Find analogies in your experience.

- What similar problems have already been solved?
- What is the analogy from known problem P' to current problem P?
- Map from known solution S' to unknown solution S

**Question:** What does this problem resemble? What's the closest solved problem I know?

### 3. Restating the Problem

Break mental ruts through re-framing.

- Change the words used to describe the problem
- Change the viewpoint entirely
- Look at it from every possible angle
- Combine multiple viewpoints simultaneously

**Question:** Can I describe this problem using completely different language or from a different perspective?

### 4. Generalization

After solving, ask: Can this go further?

- From 2 dimensions → N dimensions
- From specific field → general algebraic field
- From one problem → a class of problems

**Question:** Can I make a broader statement that includes more cases?

### 5. Structural Analysis

Break big jumps into small steps.

- Decompose into subsidiary theorems or sub-problems
- Build a path through intermediate solutions
- Often the final route can be simplified after discovery

**Question:** What smaller steps would lead from here to the solution?

### 6. Inversion

Turn the problem around.

- Swap the given and the desired
- If P → S is hard, try S → P
- Work backward from the solution

**Question:** What if the given and required were swapped? What would that path look like?

## Workflow

When user is stuck on a problem:

1. **Diagnose** — Which technique fits best?
2. **Apply** — Walk through the relevant technique(s)
3. **Iterate** — Try multiple techniques; combine insights
4. **Reflect** — After solving, can this be generalized?

## Example Prompts

- "I'm stuck on this problem..." → Apply simplification, then similar problems
- "I can't see a way forward..." → Try restating or inversion
- "How do I approach this?" → Start with simplification
- "I've been working on this forever..." → You're in a rut. Apply restating.

## Key Insight

> "It seems to be much easier to make two small jumps than the one big jump in any kind of mental thinking."

Break problems down. Connect what you know to what you need.
